<objective>
Create OSBuilder's idempotent local installer (`install.sh`) and the `.gitkeep` markers that keep the four-sub-directory layout committed across `git clone` (Pitfall 7). Per BLOCKER 1 resolution: install.sh COPIES (not symlinks) repo-root artifacts (SKILL.md, references/, scripts/state_writer.py, scripts/bootstrap.sh, scripts/bootstrap.ps1) into `${SKILL_DIR}`. Patterns are copy-pasted from `~/.claude/skills/predator/install.sh` (RESEARCH.md identifies it as the canonical idempotent installer template; predator's install.sh uses the copy pattern, NOT symlinks).

Why copy not symlink (BLOCKER 1 rationale):
- Matches `predator/install.sh` canonical pattern
- Simpler trust model for non-developers (no symlink confusion)
- Works on Windows where symlinks need admin/dev mode
- After install, `~/.claude/skills/osbuilder/SKILL.md`, `references/`, and `scripts/*` are all REAL files at the install location

Purpose: FOUND-03 requires a one-level-deep directory layout that survives clone. Without `.gitkeep` files, empty dirs vanish from git. Without `install.sh`, there's no mechanism to set up the dirs on a fresh machine. Without the COPY step, SKILL.md never reaches the install location and the Resume Protocol path in SKILL.md (`~/.claude/skills/osbuilder/scripts/state_writer.py`) is a dangling reference.

Output: 1 executable `install.sh` (idempotent, creates dirs AND copies artifacts), 3 `.gitkeep` files (`assets/`, `examples/`, `scripts/`). `references/.gitkeep` is NOT created — `references/README.md` from Plan 02 already keeps that dir committed.
</objective>

    ```sh
    #!/bin/sh
    # OSBuilder Skill Installer — Idempotent + Cross-platform
    # Source: ~/.claude/skills/predator/install.sh (paste-source per RESEARCH.md)
    # Re-running is safe; will not overwrite existing dirs and re-copies artifacts (cp -p semantics).

    set -eu

    SKILL_DIR="${HOME}/.claude/skills/osbuilder"
    # SCRIPT_DIR resolves the directory install.sh lives in (= the repo root when run
    # from a freshly-cloned checkout). All cp source paths are anchored to SCRIPT_DIR.
    SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"

    echo "[install] Installing OSBuilder skill to ${SKILL_DIR}..."
    echo "[install] Reading artifacts from ${SCRIPT_DIR}..."

    # 1. Idempotent directory creation (one level deep — Anthropic rule)
    for d in references scripts assets examples; do
        mkdir -p "${SKILL_DIR}/${d}"
    done

    # 2. .gitkeep markers for dirs that may otherwise be empty post-clone (Pitfall 7).
    #    references/ is excluded — references/README.md (copied below) already keeps it committed.
    for d in assets examples scripts; do
        if [ ! -f "${SKILL_DIR}/${d}/.gitkeep" ]; then
            : > "${SKILL_DIR}/${d}/.gitkeep"
        fi
    done

    # 3. Copy artifacts from repo root → SKILL_DIR (BLOCKER 1).
    #    Each cp is guarded so a partial repo state (e.g., before Plan 02/04/05 land)
    #    does NOT crash install — missing files print a warn line and the script continues.
    #
    #    Use cp -p to preserve mode bits; we re-chmod scripts after copying.

    # 3a. SKILL.md (Plan 02 deliverable)
    if [ -f "${SCRIPT_DIR}/SKILL.md" ]; then
        cp -p "${SCRIPT_DIR}/SKILL.md" "${SKILL_DIR}/SKILL.md"
    else
        echo "warn: ${SCRIPT_DIR}/SKILL.md not found, skipping (Plan 02 has not landed yet)" >&2
    fi

    # 3b. references/ tree (Plan 02 deliverable)
    if [ -d "${SCRIPT_DIR}/references" ]; then
        # The trailing /. copies CONTENTS into the destination dir without nesting.
        cp -Rp "${SCRIPT_DIR}/references/." "${SKILL_DIR}/references/"
    else
        echo "warn: ${SCRIPT_DIR}/references/ not found, skipping (Plan 02 has not landed yet)" >&2
    fi

    # 3c. state_writer.py (Plan 04 deliverable)
    if [ -f "${SCRIPT_DIR}/scripts/state_writer.py" ]; then
        cp -p "${SCRIPT_DIR}/scripts/state_writer.py" "${SKILL_DIR}/scripts/state_writer.py"
    else
        echo "warn: ${SCRIPT_DIR}/scripts/state_writer.py not found, skipping (Plan 04 has not landed yet)" >&2
    fi

    # 3d. bootstrap.sh + bootstrap.ps1 (Plan 05 deliverables)
    if [ -f "${SCRIPT_DIR}/scripts/bootstrap.sh" ]; then
        cp -p "${SCRIPT_DIR}/scripts/bootstrap.sh" "${SKILL_DIR}/scripts/bootstrap.sh"
    else
        echo "warn: ${SCRIPT_DIR}/scripts/bootstrap.sh not found, skipping (Plan 05 has not landed yet)" >&2
    fi
    if [ -f "${SCRIPT_DIR}/scripts/bootstrap.ps1" ]; then
        cp -p "${SCRIPT_DIR}/scripts/bootstrap.ps1" "${SKILL_DIR}/scripts/bootstrap.ps1"
    else
        echo "warn: ${SCRIPT_DIR}/scripts/bootstrap.ps1 not found, skipping (Plan 05 has not landed yet)" >&2
    fi

    # 4. Execute permissions on shipped scripts (post-copy).
    #    Silently no-op if a script is absent (cp guards above already warned).
    for f in scripts/bootstrap.sh scripts/state_writer.py; do
        if [ -f "${SKILL_DIR}/${f}" ]; then
            chmod +x "${SKILL_DIR}/${f}" || true
        fi
    done

    # 5. If Python missing, hand off to bootstrap.sh (Plan 05 deliverable).
    #    Phase 1 install.sh tolerates bootstrap.sh being absent — print a friendly
    #    note and exit 0 so the install itself stays idempotent.
    if ! command -v python3 >/dev/null 2>&1; then
        if [ -f "${SKILL_DIR}/scripts/bootstrap.sh" ]; then
            echo "[install] Python missing — invoking bootstrap.sh..."
            sh "${SKILL_DIR}/scripts/bootstrap.sh"
        else
            echo "[install] Note: python3 not found and bootstrap.sh not present yet."
            echo "[install] Run install.sh again after Plan 05 lands, or install Python 3.10+ manually."
        fi
    fi

    echo "[install] OSBuilder installed at ${SKILL_DIR}"
    echo "[install] Try: /osbuilder help (after Phase 3 lands)"
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user shell ↔ install.sh | User invokes `bash install.sh` (or `sh install.sh`) — script must not assume cwd, must not write outside `${SKILL_DIR}`, must not require root |
| repo root ↔ install.sh source paths | Read paths anchored to `${SCRIPT_DIR}=$(cd $(dirname $0) && pwd)` — script reads only files inside the repo it was cloned into; never follows arbitrary `${PWD}/...` references |
| install.sh ↔ bootstrap.sh | install.sh hands off to bootstrap.sh ONLY when Python missing AND bootstrap.sh exists in `${SKILL_DIR}` post-copy — never blindly invokes a missing script |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-V14 | Tampering — destructive ops | install.sh | mitigate | NO `rm -rf`, NO `sudo` in install.sh; `mkdir -p` is non-destructive (existing dirs untouched); `.gitkeep` creation guarded by `[ ! -f ]` (no clobber); `cp -p` overwrites are intentional (re-install latest artifact) |
| T-1-V12 | Tampering — path traversal via cwd-relative paths | install.sh | mitigate | All write paths anchored to `${SKILL_DIR}="${HOME}/.claude/skills/osbuilder"` and all read paths anchored to `${SCRIPT_DIR}="$(cd "$(dirname "$0")" && pwd)"` — script never uses bare `./assets` or `${PWD}/...` (which could be tampered if user runs from a malicious cwd) |
| T-1-V7 | Information Disclosure — raw script errors leak to user | install.sh | mitigate | `set -eu` causes early exit with clean shell error message; missing-source warnings prefixed `warn:` are friendly + non-fatal; no Python tracebacks possible (this is shell, not Python) |
| T-1-Pitfall-7 | Loss-of-data — empty dirs disappear from git | assets/, examples/, scripts/ | mitigate | `.gitkeep` files committed in source repo; install.sh ALSO creates them at runtime in `${SKILL_DIR}` for direct-clone-to-skill-dir installs |
| T-1-V14 | Idempotency violation | install.sh second run | mitigate | Every operation is idempotent: `mkdir -p` is no-op-if-exists; `.gitkeep` creation guarded by file-existence check; `chmod +x` is no-op-if-already-executable; `cp -p` overwrite is the intended semantic for re-running install (latest artifact wins) |
| T-1-BLOCKER-1 | Repudiation — SKILL.md never reaches install location | install.sh | mitigate | Explicit `cp -p ${SCRIPT_DIR}/SKILL.md ${SKILL_DIR}/SKILL.md` block, source-guarded, with warn-on-missing semantics. Acceptance gate: `test -f ~/.claude/skills/osbuilder/SKILL.md` post-install. |
