The wrapper is intentionally minimal. Do NOT add argparse, do NOT add logging, do NOT add `--dry-run` (use `python scripts/preflight_check.py uninstall` if a flag is needed later — wrappers stay scope-locked).

```python
#!/usr/bin/env python3
"""uninstall.py — remove anything OSBuilder's preflight installed (PRE-06).

Thin wrapper around preflight_check.uninstall(). Exists so `/osbuilder uninstall`
maps to a script with that exact name, mirroring state_writer.py's one-CLI-per-script
pattern. The actual algorithm (read install-log.json, walk in reverse, run each
recorded uninstall_argv) lives in preflight_check.py — see Plan 02-02.
"""
import sys
from pathlib import Path

# Ensure scripts/ is importable when invoked directly (not via pytest)
sys.path.insert(0, str(Path(__file__).parent))
from preflight_check import uninstall  # noqa: E402

if __name__ == "__main__":
    raise SystemExit(uninstall())
```

`sys.path.insert(0, str(Path(__file__).parent))` is needed because:
- When `python scripts/uninstall.py` is invoked, Python's CWD-based search may not include `scripts/`.
- pytest's `pythonpath = ["scripts"]` (from `pyproject.toml`) only applies during pytest runs.
- The shim ensures the script works in both contexts.

Test stubs (already in `scripts/tests/test_uninstall.py` from Plan 02-01) call `un.uninstall()` where `un = importlib.import_module("preflight_check")` — they exercise the FUNCTION, not the wrapper SCRIPT. So `test_uninstall_reverses_all` and `test_uninstall_preserves_pre_existing` flip GREEN as soon as `preflight_check.uninstall` works (which it does after Plan 02-02). This plan adds the SCRIPT entry point.
