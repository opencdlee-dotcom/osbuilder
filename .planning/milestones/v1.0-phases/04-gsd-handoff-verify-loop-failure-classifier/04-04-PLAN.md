Core verify_npm pattern (RESEARCH.md Pattern 4 — verified against live npm):
```python
def verify_npm(package_name: str, timeout: int = 10) -> bool:
    url = f"https://registry.npmjs.org/{package_name}"
    req = urllib.request.Request(url, method="HEAD")
    try:
        with urllib.request.urlopen(req, timeout=timeout) as resp:
            return resp.status == 200
    except urllib.error.HTTPError as e:
        return e.code != 404  # 404 = not found; other 4xx = network issue, not hallucinated
    except (urllib.error.URLError, OSError):
        return True  # fail-open: network error ≠ hallucinated package
```

CLI exit codes:
- Exit 0: package verified to exist on registry
- Exit 1: package does NOT exist (hallucinated) — block install

def fake_urlopen(req, timeout=None):
    raise urllib.error.HTTPError(
        url=req.full_url, code=404, msg="Not Found", hdrs=None, fp=None
    )
monkeypatch.setattr("urllib.request.urlopen", fake_urlopen)

# HTTP 200 → verify_npm returns True
class FakeResponse:
    status = 200
    def __enter__(self): return self
    def __exit__(self, *a): pass
monkeypatch.setattr("urllib.request.urlopen", lambda *a, **kw: FakeResponse())

# URLError → verify_npm returns True (fail-open)

)

Create `scripts/registry_verify.py`. Implementation rules:

1. **Header and imports** — stdlib only:
   ```python
   #!/usr/bin/env python3
   """registry_verify.py — OSBuilder package registry existence gate.

   Checks whether a package name exists on a public registry (npm, PyPI,
   crates.io) before any install command runs. Defends against slopsquatting
   (hallucinated package names) without downloading the package.

   Exit codes:
     0 = package verified to exist (or network error — fail-open)
     1 = package does NOT exist on registry — BLOCK INSTALL

   Pure stdlib — no third-party deps.
   """
   from __future__ import annotations

   import argparse
   import sys
   import urllib.error
   import urllib.request
   ```

2. **verify_npm()** — exact implementation from interface block. Use `method="HEAD"` to avoid downloading the full package metadata JSON.

3. **verify_pypi()** — same shape, different URL:
   ```python
   def verify_pypi(package_name: str, timeout: int = 10) -> bool:
       url = f"https://pypi.org/pypi/{package_name}/json"
       req = urllib.request.Request(url, method="HEAD")
       try:
           with urllib.request.urlopen(req, timeout=timeout) as resp:
               return resp.status == 200
       except urllib.error.HTTPError as e:
           return e.code != 404
       except (urllib.error.URLError, OSError):
           return True  # fail-open
   ```

4. **verify_cargo()** — same shape, crates.io endpoint:
   ```python
   def verify_cargo(package_name: str, timeout: int = 10) -> bool:
       url = f"https://crates.io/api/v1/crates/{package_name}"
       req = urllib.request.Request(url, method="HEAD")
       # crates.io requires a User-Agent header
       req.add_header("User-Agent", "OSBuilder/1.0 (registry verification gate)")
       try:
           with urllib.request.urlopen(req, timeout=timeout) as resp:
               return resp.status == 200
       except urllib.error.HTTPError as e:
           return e.code != 404
       except (urllib.error.URLError, OSError):
           return True  # fail-open
   ```

5. **main() with argparse** — `--pkg` (required) + `--ecosystem` (choices: npm, pip, cargo, required):
   ```python
   def main(argv: list[str] | None = None) -> int:
       if argv is None:
           argv = sys.argv[1:]
       parser = argparse.ArgumentParser(
           prog="registry_verify",
           description="OSBuilder package registry existence gate.",
       )
       parser.add_argument("--pkg", required=True,
                           help="Package name to verify")
       parser.add_argument("--ecosystem", choices=("npm", "pip", "cargo"), required=True,
                           help="Package ecosystem to check")
       args = parser.parse_args(argv)

       verifiers = {"npm": verify_npm, "pip": verify_pypi, "cargo": verify_cargo}
       exists = verifiers[args.ecosystem](args.pkg)
       if not exists:
           sys.stderr.write(
               f"OSBuilder: package '{args.pkg}' not found on {args.ecosystem} registry. "
               f"Install blocked (slopsquatting defense).\n"
           )
           return 1
       return 0

   if __name__ == "__main__":
       raise SystemExit(main())
   ```

6. **No shell=True, no subprocess, no third-party libraries.** The only network call is via `urllib.request.urlopen`.

7. **HTTPError handling subtlety:** `e.code != 404` returns True for e.g. HTTP 429 (rate limit) — this is intentional. We only block installs when the registry explicitly says "not found." Rate limits and server errors are treated as "we don't know" → fail-open.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Tampering | Shell injection via --pkg arg | mitigate | registry_verify.py has no subprocess calls — package name is only interpolated into a URL string for urllib.request, not into a shell command; URL string injection is mitigated by urllib encoding |
| T-04-04-02 | Tampering | SSRF via crafted package name (e.g., internal host) | accept | Package names are constrained to npm/pip/cargo naming conventions by the registries themselves; registry_verify.py calls only hardcoded registry base URLs (npmjs.org, pypi.org, crates.io); a caller passing a crafted name would only hit those endpoints |
| T-04-04-03 | Denial of Service | Build blocked by registry downtime | mitigate | Fail-open policy: URLError returns True (package assumed to exist); only explicit HTTP 404 blocks install; timeout=10 seconds caps any hanging call |
| T-04-04-04 | Denial of Service | Build blocked by slow registry HEAD response | mitigate | timeout=10 parameter to urlopen; OSError caught alongside URLError |
| T-04-04-05 | Spoofing | Man-in-the-middle on registry HTTPS | accept | urllib.request uses system TLS; HTTPS certificate validation applies; MITM would require a compromised system cert — outside OSBuilder's threat scope for v1 |
