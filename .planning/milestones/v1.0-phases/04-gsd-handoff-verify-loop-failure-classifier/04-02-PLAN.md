```python
def _resolve_project_root(arg: str | None) -> Path:
    if arg is None:
        cur = Path.cwd().resolve()
        for parent in (cur, *cur.parents):
            if (parent / ".planning").is_dir():
                return parent
        return cur
    if ".." in arg:
        raise SystemExit("OSBuilder: --project-root cannot contain '..' segments.")
    return Path(arg).resolve()
```

```python
def atomic_write(path: Path, content: str) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    tmp = path.with_name(f".{path.stem}.{os.getpid()}.tmp")
    try:
        tmp.write_text(content, encoding="utf-8")
        os.replace(str(tmp), str(path))
    except BaseException:
        tmp.unlink(missing_ok=True)
        raise
```

Create `scripts/gsd_driver.py` following the scaffold_dispatch.py patterns documented in PATTERNS.md and the interface block above. Key implementation rules:

1. **Never use subprocess for slash commands.** `emit_next_command` prints slash commands to stdout via `print()`. Only use subprocess for: `state_writer.py` (state reads/writes) and `registry_verify.py` (install gate).

2. **Always read state first.** Entry point reads state.md via `state_writer.py read --format json` before any other action. Never initialize `retry_count`, `phase_step`, or `current_phase` from Python defaults.

3. **Read state via subprocess:**
   ```python
   def _read_state(project_root: Path) -> dict:
       state_path = project_root / ".planning" / "osbuilder" / "state.md"
       result = subprocess.run(
           [sys.executable, str(STATE_WRITER), "read", "--format", "json",
            "--project-root", str(project_root)],
           capture_output=True, text=True, shell=False, check=True,
       )
       return json.loads(result.stdout)
   ```

4. **PHASE_STEP_COMMANDS dispatch** (from interface block above). Steps 2, 7, 9 are handled in-line (registry gate, VERIFICATION.md write, phase advance) — they don't emit slash commands.

5. **Phase advance at step 9:** Write `phase_step=0` and `current_phase=N+1` to state.md via state_writer write subcommand.

6. **Escalation at retry_count >= 3:** Before dispatching the normal phase_step command, check `int(state.get("retry_count", "0"))`. If >= 3, print `/gsd-debug` then print `/problem-solver` and return — do not emit the normal step command.

   ```python
   def build_install_cmd(package: str, ecosystem: str) -> list[str]:
       """Build a verified-package install command with --ignore-scripts."""
       if ecosystem == "npm":
           return ["npm", "install", "--ignore-scripts", package]
       if ecosystem == "pip":
           return ["pip", "install", "--no-deps", package]
       if ecosystem == "cargo":
           return ["cargo", "add", package]
       raise ValueError(f"Unknown ecosystem: {ecosystem}")
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| state.md → gsd_driver.py | state.md fields (current_phase, retry_count, phase_step) come from disk; integer parsing must be guarded |
| gsd_driver.py → registry_verify.py subprocess | package name from state_writer flows into subprocess args |
| gsd_driver.py → state_writer.py subprocess | values written to state.md pass through _check_value_safe in state_writer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | state.md integer fields (current_phase, retry_count, phase_step) | mitigate | Wrap `int(state.get("field", "0"))` in try/except ValueError; default to 0 on parse failure; never eval() the value |
| T-04-02-02 | Tampering | Prompt injection via error strings emitted to stdout | mitigate | gsd_driver.py prints only hardcoded slash command strings from PHASE_STEP_COMMANDS dict — never interpolates raw error strings into emitted slash commands |
| T-04-02-03 | Tampering | Shell injection via package name in install command | mitigate | build_install_cmd returns list form for subprocess.run(shell=False); package name is never passed through shell=True or string interpolation |
| T-04-02-04 | Tampering | Path traversal in VERIFICATION.md write | mitigate | VERIFICATION.md path is constructed as project_root / ".planning" / "osbuilder" / "VERIFICATION.md" — uses _resolve_project_root with ".." guard; atomic_write via os.replace |
| T-04-02-05 | Denial of Service | Infinite retry if retry_count not read from state.md | mitigate | emit_next_command always calls _read_state() first; retry_count check before any dispatch; cap enforced at >= 3 |
</threat_model>
