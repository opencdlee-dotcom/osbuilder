    global _TUTOR_ENABLED, _MODE
    try:
        result = subprocess.run(

Implement thread-per-stream subprocess capture (copy from PATTERNS.md lines 112–151 verbatim):
- `_drain_stream(stream, lines: list[str], log_handle) -> None`
- `capture_subprocess(cmd: list[str], role: str, action: str, *, log_path: Path, cwd: Path | None = None, timeout: float | None = None) -> tuple[int, list[str], list[str]]`

The capture_subprocess function:
1. Calls `emit(role, action, "start", detail=" ".join(cmd[:2]))` before starting the subprocess.
2. Opens log_path in append mode (`"a"` encoding="utf-8").
3. Writes a header line: `f"\n# {_now_iso()} {role} {action}: {' '.join(cmd)}\n"`.
4. Uses subprocess.Popen with stdout=PIPE, stderr=PIPE, bufsize=1, text=True, shell=False.
5. Starts two threads: t_out drains proc.stdout, t_err drains proc.stderr; both append to lines and write to log.
6. Calls proc.wait(timeout=timeout); on TimeoutExpired → proc.kill() + proc.wait().
7. Joins both threads.
8. If returncode == 0: emit(role, action, "ok"). Else: emit(role, action, "fail", detail="(see debug log)").
9. Returns (returncode, stdout_lines, stderr_lines).

Helper: `_now_iso() -> str` = `datetime.datetime.now(datetime.timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| subprocess stdout/stderr → narration capture | Raw subprocess output crosses here; narration must treat it as data only (never as instructions) |
| role brief Markdown → _parse_brief_markdown | Attacker-controlled brief content could include malformed sections; parser must not crash on malformed input |
| state.md mode/tutor_enabled → narration._refresh_state | State is written by state_writer (validated); narration reads it via subprocess, not direct file parse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Information Disclosure | capture_subprocess raw output → build.log | accept | build.log is not committed (convention from Phase 1); may contain tokens/keys from subprocess environment; documented in README; gitignore in Phase 6 |
